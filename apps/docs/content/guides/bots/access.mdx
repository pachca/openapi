# Доступы к чатам и сообщениям

Бот взаимодействует с пространством как участник с ролью **«Сотрудник»** — он видит открытые каналы и сообщения в них, включая треды.

## Закрытые каналы и беседы

Чтобы бот получил доступ к закрытому каналу или беседе, его нужно добавить туда как участника. После добавления бот автоматически получает доступ ко всем тредам этого чата.

Добавить бота в чат можно двумя способами:
- Через интерфейс — в диалоге **Добавление участников** на вкладке **Интеграции**. Список доступных ботов зависит от настройки [Кто может добавлять бота в чаты](/guides/bots/setup#kto-mozhet-dobavlyat-bota-v-chaty).
- Через API — метод [Добавление участника в чат](POST /chats/{id}/members), передав `user_id` бота в `member_ids`.

## Треды

Бота можно добавить напрямую в тред — без добавления в родительский чат. Это возможно благодаря [сквозным тредам](https://pachca.com/blog/chto-takoe-skvoznye-tredy-i-kak-ih-ispolzovat) Пачки: участник треда видит только сам тред и родительское сообщение, но не историю чата.

Добавить бота в тред можно двумя способами:
- Через API — метод [Добавление участника в чат](POST /chats/{id}/members), передав `chat_id` треда
- Через упоминание `@никнейм_бота` в сообщении — доступно, когда бот настроен как **публичный**

## Личные сообщения

Пользователь может писать боту личные сообщения и взаимодействовать с ним 1 на 1 в двух случаях:

- Бот настроен как **публичный** — любой участник пространства может начать диалог
- Бот сам ранее [отправил сообщение](POST /messages) пользователю через API (с `entity_type: "user"`) — после этого пользователь может отвечать, даже если бот не публичный

При этом бота нельзя добавить в существующие личные сообщения между двумя участниками пространства.

<Info>
  Бот получает [исходящие вебхуки](/guides/webhook) только из тех чатов и тредов, в которых он состоит. Глобальные события (например, изменение участников пространства) приходят без добавления в чат.
</Info>
