
# Боты

Бот в Пачке — это посредник между вашим сервисом и мессенджером. Он может отправлять и получать сообщения, реагировать на команды пользователей, открывать интерактивные формы и создавать предпросмотр ссылок.

## Создание бота

Для создания бота перейдите в **«Автоматизации»** → **«Интеграции»** → **«Чат-боты и Вебхуки»** и нажмите кнопку **«+»**.


  ### Шаг 1. Укажите имя и логотип

Имя бота будет отображаться в сообщениях. Загрузите логотип — он будет использоваться как аватар бота.


  ### Шаг 2. Выберите область использования

Бот может работать в одном конкретном чате или в нескольких. Для работы в нескольких чатах бота нужно добавить в каждый из них.


  ### Шаг 3. Сохраните access_token

После создания бота скопируйте `access_token` — он потребуется для работы с API от имени бота.


> Подробнее о создании и настройке ботов вы можете прочитать в статье
  [Чат-боты через Webhook/API](https://www.pachca.com/articles/webhook)


## Настройки бота

### Управление доступом

Настройте, кто может редактировать бота — только создатель или все администраторы пространства.

### Разрешения на добавление

Определите, кто может добавлять бота в чаты — создатель, администраторы или все сотрудники.

### Видимость

- **Публичный бот** — доступен в глобальном поиске, можно отправить ЛС
- **Приватный бот** — доступен только тем, кто имеет к нему доступ

### Редактирование через API

Вы можете обновить настройки бота программно — метод [Обновить бота](/api/bots/update).

## Возможности бота

### Входящие вебхуки

Отправка сообщений в чаты от имени бота без использования API — достаточно одного HTTP-запроса. Подробнее — в разделе [Входящие вебхуки](/guides/incoming-webhooks).

### Исходящий вебхук

Получение уведомлений о событиях в реальном времени (новые сообщения, реакции, нажатия кнопок и другие). Подробнее — в разделе [Исходящий вебхук](/guides/webhook).

### Формы

Интерактивные модальные окна с полями ввода, которые открываются по нажатию кнопки в сообщении бота. Подробнее — в разделе [Формы](/guides/forms).

### API

Полный доступ к API Пачки: создание чатов, управление участниками, отправка сообщений с кнопками и файлами. Справочник методов доступен в разделе [API Reference](/api/quickstart).

## Кнопки

Под сообщениями бота можно размещать интерактивные кнопки. Кнопки задаются как массив строк, каждая строка — массив объектов кнопок.

### Типы кнопок

| Тип | Описание |
|-----|----------|
| **URL** | Открывает ссылку в браузере. Поле `url` содержит адрес |
| **Data** | Отправляет данные на сервер через исходящий вебхук. Поле `data` содержит произвольную строку |

### Ограничения

- Максимум **8 кнопок** в одной строке
- Максимум **100 кнопок** на одно сообщение
- Текст кнопки — до **255 символов**

### Получение нажатий

При нажатии `Data`-кнопки вашему приложению отправляется исходящий вебхук с информацией о нажатии. После получения вы можете [отредактировать сообщение](PUT /messages/{id}) (например, обновить кнопки) или [отправить новое сообщение](POST /messages) в ответ.

Подробнее о кнопках — в статье [Кнопки в чат-ботах](https://www.pachca.com/articles/knopki-v-chat-botah)

## Предпросмотр ссылок (Unfurl)

Unfurl позволяет создавать кастомный предпросмотр для ссылок, которые отправляют пользователи в чатах.

### Как это работает

1. Администратор создаёт **Unfurl-бота** и указывает до **5 доменов** для отслеживания
2. Когда пользователь отправляет сообщение со ссылкой на один из доменов, Пачка отправляет вебхук
3. Бот вызывает API-метод [Unfurl (разворачивание ссылок)](/api/link-previews/add) для создания предпросмотра

### Правила доменов

- Поддомены отслеживаются автоматически (указав `example.com`, вы получите вебхуки для `app.example.com`)
- IP-адреса не поддерживаются — только доменные имена

> Подробнее о разворачивании ссылок — в статье
  [Unfurling ссылок в Пачке](https://www.pachca.com/articles/unfurling-ssylok-v-pachce)


## Готовые open-source боты

### Welcome Bot

Автоматическое приветствие новых сотрудников в личных сообщениях. Три шаблона сообщений: `short`, `default`, `extended`. Развёртывание на Vercel или собственном сервере.

> Подробнее — в статье [Welcome Bot](https://www.pachca.com/blog-posts/welcome-bot)


### Review Bot

Синхронизация GitHub Pull Requests с чатом Пачки: уведомления о новых PR, статусы проверок, review, workflow runs, авто-упоминание пользователей. Развёртывание на Яндекс.Облаке или собственном сервере.

> Подробнее — на [GitHub](https://github.com/pachca/review-bot)
