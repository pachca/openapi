# Блоки представления

Блоки - это компоненты представления, которые позволяют вам формировать необходимые поля ввода данных и отображать дополнительную пояснительную информацию. Они указываются в поле `view.blocks` при открытии представления. Вы можете добавить до **100 блоков** в одно представление.

## Заголовок (header)

![Заголовок](/images/forms/header.webp)

*Заголовок*


#### ViewBlockHeader

- `type: string` (required) — Тип блока. Пример: `"header"`
  Значения: `header` — Для заголовков всегда header
- `text: string` (required, max length: 150) — Текст заголовка. Пример: `"Основная информация"`


## Обычный текст (plain_text)

![Обычный текст](/images/forms/plaintext.webp)

*Обычный текст*


#### ViewBlockPlainText

- `type: string` (required) — Тип блока. Пример: `"plain_text"`
  Значения: `plain_text` — Для обычного текста всегда plain_text
- `text: string` (required, max length: 12000) — Текст. Пример: `"Заполните форму. После отправки формы в общий чат будет отправлено текстовое уведомление, а ваш отпуск будет сохранен в базе."`


## Форматированный текст (markdown)

![Форматированный текст](/images/forms/markdown.webp)

*Форматированный текст*


#### ViewBlockMarkdown

- `type: string` (required) — Тип блока. Пример: `"markdown"`
  Значения: `markdown` — Для форматированного текста всегда markdown
- `text: string` (required, max length: 12000) — Текст. Пример: `"Информацию о доступных вам днях отпуска вы можете прочитать по [ссылке](https://www.website.com/timeoff)"`


В поле `text` вам необходимо прислать форматированный текст в синтаксисе `markdown`. Полный список поддерживаемых конструкций — в разделе [Форматирование текста](/guides/markdown). Списки и цитаты в блоках формы не распознаются и отображаются обычным текстом.

## Текстовое поле (input)

![Текстовое поле](/images/forms/input.webp)

*Текстовое поле*


#### ViewBlockInput

- `type: string` (required) — Тип блока. Пример: `"input"`
  Значения: `input` — Для текстового поля всегда input
- `name: string` (required, max length: 255) — Название, которое будет передано в ваше приложение как ключ указанного пользователем значения. Пример: `"info"`
- `label: string` (required, max length: 150) — Подпись к полю. Пример: `"Описание отпуска"`
- `placeholder: string` (max length: 150) — Подсказка внутри поля ввода, пока оно пустое. Пример: `"Куда собираетесь и что будете делать"`
- `multiline: boolean` (default: false) — Многострочное поле. Пример: `true`
- `initial_value: string` (max length: 3000) — Начальное значение в поле. Пример: `"Начальный текст"`
- `min_length: integer, int32` (min: 0, max: 3000) — Минимальная длина текста, который должен написать пользователь. Если пользователь напишет меньше, он получит ошибку.. Пример: `10`
- `max_length: integer, int32` (min: 1, max: 3000) — Максимальная длина текста, который должен написать пользователь. Если пользователь напишет больше, он получит ошибку.. Пример: `500`
- `required: boolean` (default: false) — Обязательность. Пример: `true`
- `hint: string` (max length: 2000) — Подсказка, которая отображается под полем серым цветом. Пример: `"Возможно вам подскаджут, какие места лучше посетить"`


## Выпадающий список (select)

![Выпадающий список](/images/forms/select.webp)

*Выпадающий список*


#### ViewBlockSelect

- `type: string` (required) — Тип блока. Пример: `"select"`
  Значения: `select` — Для выпадающего списка всегда select
- `name: string` (required, max length: 255) — Название, которое будет передано в ваше приложение как ключ указанного пользователем выбора. Пример: `"team"`
- `label: string` (required, max length: 150) — Подпись к выпадающему списку. Пример: `"Выберите команду"`
- `options: array of object` (max items: 100) — Массив доступных пунктов в выпадающем списке
  - `text: string` (required, max length: 75) — Отображаемый текст. Пример: `"Ничего"`
  - `value: string` (required, max length: 150) — Уникальное строковое значение, которое будет передано в ваше приложение при выборе этого пункта. Пример: `"nothing"`
  - `selected: boolean` — Изначально выбранный пункт. Только один пункт может быть выбран.. Пример: `true`
- `required: boolean` (default: false) — Обязательность. Пример: `false`
- `hint: string` (max length: 2000) — Подсказка, которая отображается под выпадающим списком серым цветом. Пример: `"Выберите одну из команд"`

## Загрузка файлов (file_input)

![Загрузка файлов](/images/forms/file.webp)

*Загрузка файлов*


#### ViewBlockFileInput

- `type: string` (required) — Тип блока. Пример: `"file_input"`
  Значения: `file_input` — Для загрузки файлов всегда file_input
- `name: string` (required, max length: 255) — Название, которое будет передано в ваше приложение как ключ указанного пользователем значения. Пример: `"request_doc"`
- `label: string` (required, max length: 150) — Подпись к полю. Пример: `"Заявление"`
- `filetypes: array of string` — Массив допустимых расширений файлов, указанные в виде строк (например, ["png","jpg","gif"]). Если это поле не указано, все расширения файлов будут приняты.. Пример: `["pdf","jpg","png"]`
- `max_files: integer, int32` (default: 10, min: 1, max: 10) — Максимальное количество файлов, которое может загрузить пользователь в это поле.. Пример: `1`
- `required: boolean` (default: false) — Обязательность. Пример: `true`
- `hint: string` (max length: 2000) — Подсказка, которая отображается под полем серым цветом. Пример: `"Загрузите заполненное заявление с электронной подписью (в формате pdf, jpg или png)"`
