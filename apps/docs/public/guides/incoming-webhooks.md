
# Входящие вебхуки

Входящий вебхук позволяет отправлять сообщения в чаты Пачки от имени бота — достаточно одного HTTP POST-запроса. Вам не нужно работать с API, а гибкость обработки данных обеспечивают шаблонизаторы.

## Как это работает


  ### Шаг 1. Скопируйте Webhook URL

В настройках бота перейдите на вкладку **«Входящий webhook»** и скопируйте сгенерированный URL.


  ### Шаг 2. Вставьте URL в настройки внешнего сервиса

Укажите скопированный URL как адрес для уведомлений (webhook URL) в вашей CRM, CI/CD, мониторинге или другом сервисе.


  ### Шаг 3. Сервис отправляет POST-запрос

При наступлении события сервис отправляет `POST`-запрос с JSON-данными на Webhook URL.


  ### Шаг 4. Сообщение появляется в чате

Пачка обрабатывает запрос и публикует сообщение от имени бота во все групповые чаты, где он состоит.


## Минимальный формат запроса

```json title="POST на Webhook URL"
{
  "message": "Текст сообщения"
}
```

Поле `message` содержит текст, который будет опубликован от имени бота.

## Шаблоны сообщений

Если внешний сервис отправляет JSON с данными (а не готовый текст), вы можете использовать **шаблонизаторы** для форматирования сообщения. Пачка поддерживает два шаблонизатора:

- [**Liquid**](https://shopify.github.io/liquid/) — мощный шаблонизатор от Shopify с условиями, циклами и фильтрами
- [**Mustache**](https://mustache.github.io/) — минималистичный шаблонизатор без логики

Шаблон настраивается в интерфейсе Пачки в настройках бота → **«Входящий webhook»** → **«Шаблон сообщения»**.

### Пример

Если сервис отправляет JSON:

```json title="Входящий JSON от CRM"
{
  "client": "Иван Петров",
  "amount": 15000,
  "status": "оплачен"
}
```

Шаблон Liquid:

```text title="Шаблон сообщения"
Заказ от {{ client }} на сумму {{ amount }} ₽ — {{ status }}
```

Результат в чате: `Заказ от Иван Петров на сумму 15000 ₽ — оплачен`

## Ограничения

> **Внимание:** Входящий вебхук не позволяет отправить личное сообщение или указать конкретный чат. Сообщение
  публикуется во **все групповые чаты**, где состоит бот. Для отправки в конкретный чат или ЛС
  используйте API-метод [Новое сообщение](POST /messages).
