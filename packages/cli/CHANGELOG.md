# Changelog

## 2026.3.9  (19 марта 2026)

- **Изменено** (all): Убрана клиентская валидация скоупов — проверка разрешений теперь только на стороне API (403)
- **Изменено** (auth login): Профиль больше не сохраняет список скоупов — только тип, имя и email
- **Удалено** (auth refresh): Команда удалена — кеширование скоупов больше не используется

## 2026.3.8  (17 марта 2026)

- **Добавлено** (upgrade): Команда pachca upgrade — обновление CLI до последней версии
- **Удалено** (init hook): Ошибки авторизации теперь выводятся в JSON при -o json (раньше формат игнорировался)
- **Удалено** (update banner): Баннер обновления теперь показывается даже при ошибках команд

## 2026.3.7  (16 марта 2026)

- **Добавлено** (upload): Загрузка файла одной командой (получает подпись + загружает автоматически)
- **Изменено** (common direct-url): Обязательный флаг --direct-url вместо хардкод URL, запрос идёт без Authorization

## 2026.3.6  (15 марта 2026)

- **Изменено** (threads): Топик thread переименован в threads (pachca threads add, pachca threads get)

## 2026.3.5  (7 марта 2026)

- **Изменено** (guide): Сценарии переписаны на CLI-примеры с подсказками вместо описаний API-методов
- **Добавлено** (views open): Пример «Форма заявки/запроса»

## 2026.3.4  (5 марта 2026)

- **Добавлено** (all): Переменная окружения PACHCA_API_URL для использования кастомного base URL
- **Изменено** (all): Исправлено выравнивание в уведомлении о доступном обновлении
